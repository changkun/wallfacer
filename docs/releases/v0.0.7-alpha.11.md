## 🚀 v0.0.7-alpha.11 — The "Plan in Vue" Release

**Release date:** 2026-05-30
**Previous release:** v0.0.7-alpha.10 (2026-05-28)
**Range:** v0.0.7-alpha.10..v0.0.7-alpha.11
**Delta:** 11 commits · 58 files changed · +5,238 / −6,207

The Plan/Spec experience finally lands on the new Vue frontend with full feature parity, the roadmap gets a sharp rescope around the Latere platform boundary, and the runner stops face-planting when your configured env file isn't there.

### 🧭 Plan & Spec Page on Vue

- **Full Plan page port** — `frontend/src/views/PlanPage.vue` now hosts the planning chat, spec tree, and focused spec view as first-class Vue components, replacing the legacy `ui/` shell with `PlanningChatPanel.vue`, `SpecTreePanel.vue`, `SpecFocusedView.vue`, and a sticky `FloatingToc.vue`.
- **Streaming chat as a composable** — the new `useStreamingFetch` composable and `stores/planning.ts` Pinia store cleanly model in-flight turns, thread switching, undo, and interrupt without the old DOM glue.
- **Mermaid + markdown rendering** — `lib/markdown.ts`, `lib/mermaidRender.ts`, and `lib/specFrontmatter.ts` give specs rich rendering with diagrams, frontmatter parsing, and pretty NDJSON for streamed agent output.
- **First-spec toast no longer loops** — the spec-explorer bootstrap remembers it has already nudged you, so refreshing the page stops replaying the welcome toast.

### 🎨 Theme & Branding Cleanup

- **Theme consolidated into prefs** — `useTheme` is gone; theme state now lives on the `prefs` store, simplifying hydration and removing a duplicated source of truth.
- **Latere logo mark variants** — new logo assets, covered by `latereLogo.test.ts`, give the app a consistent mark across light/dark surfaces.
- **Marketing pricing page removed** — `PricingPage.vue`, its route, nav entry, and i18n strings are gone; wallfacer ships as a tool, not a storefront.

### 🛠 Runner Robustness

- **Graceful env-file fallback** — when the configured env file path is missing, the runner now falls back to the default `~/.wallfacer/.env` instead of erroring out, with `internal/runner/container_builder_test.go` locking in the behavior.
- **Container builder refactor** — `internal/runner/container.go` cleanly threads env-file resolution through the build pipeline, covered by new unit tests rather than relying on E2E to catch regressions.
- **Execute handler tests** — `internal/handler/execute_handler_test.go` adds coverage for the launch path that previously only had implicit coverage from E2E.

### 🔁 Routine Cleanup Semantics

- **Cancel/archive clears `enabled`** — cancelling or archiving a routine task now clears its `RoutineEnabled` flag, so a deactivated routine stays deactivated and doesn't re-fire on the next scheduler tick.
- **Documented** — `docs/guide/board-and-tasks.md` and `docs/internals/workspaces-and-config.md` spell out the new contract so operators don't get surprised.

### 🗺 Roadmap Rescope: Latere Boundary

- **Cloud track aligned to Latere** — `specs/cloud/cloud-infrastructure.md` is rescoped to **server-deploy**, with sandbox runtime handed off to the new **Cella runtime** spec at `specs/cloud/latere-integration/cella-runtime.md`.
- **New `latere-integration.md`** — captures the platform boundary between wallfacer and Latere, including identity, billing idempotency, and tenant API touch points.
- **Strategic specs relocated** — out-of-scope research specs (multi-agent debate/consensus, defense-in-depth, eval pipeline, intelligence system, agent-memory-identity, telemetry-observability, audit-log, information-inbox, sandbox-hooks, oversight-risk-scoring) move out of the active roadmap, sharpening `specs/README.md` to what's actually being built.
- **Rebrand-module-path scope refreshed** — the in-flight rename gets its image-scope updated to match the new boundary.

### 📋 Operator Notes

- **No config or API changes required.** The env-file fallback is purely additive.
- **Routine behavior change:** cancel/archive now clears the routine's `enabled` flag. If you had tooling that re-enabled cancelled routines on the next tick, it will now need to flip `enabled` back explicitly.
- **Spec tree reshuffle:** if you were tracking specs under `specs/observability/`, `specs/oversight/`, `specs/intelligence/`, or `specs/shared/{eval-pipeline,information-inbox}`, those have been removed from the active roadmap. Check git history for the originals.
