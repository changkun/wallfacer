### Two weeks. Half a thousand commits. 100K lines. This is not a point release.

Wallfacer v0.0.6 is the biggest update yet — a ground-up hardening of everything that makes autonomous task execution actually *reliable*. New platform support, a completely rebuilt UI, deep observability, and an architecture that's ready for what comes next.

## 🪟 Windows & WSL2 — Wallfacer Runs Everywhere Now

- **Full Windows support** via WSL2 — build tags, path handling, signal management, and line endings all work natively
- **Cross-platform CI** with dedicated Linux, macOS, and Windows test jobs
- Platform-specific fixes for temp directories, path separators, shell quoting, and timing-sensitive tests
- `.gitattributes` enforcing LF line endings across all platforms

## 📦 One-Liner Installation

No more cloning and building from source. Just run:

```bash
curl -fsSL https://raw.githubusercontent.com/changkun/wallfacer/main/install.sh | sh
```

Pre-built binaries for **linux/amd64**, **linux/arm64**, **darwin/amd64**, **darwin/arm64**, and **windows/amd64** are attached to this release. The install script auto-detects your OS and architecture.

## 🩺 `wallfacer doctor` — Preflight Health Check

New `wallfacer doctor` command validates your entire setup before you run:

- Checks container runtime availability
- Validates API credentials
- Verifies sandbox images are cached
- Reports missing dependencies with actionable fix suggestions

## 🏗️ Workspace Groups — Multi-Project Orchestration

Wallfacer now manages **multiple workspace groups** simultaneously:

- **Tab-strip UI** with VS Code-style tabs — switch between workspace groups instantly
- **Persistent workspace groups** that survive restarts
- **Workspace browser** with directory picker, dotfolder hiding, and path filtering
- **Per-workspace usage attribution** — see exactly what each workspace costs
- **Per-workspace git status** with parallelized status collection
- Workspace pills with truncated paths and `~` shorthand

## 📖 In-App Documentation Viewer

The full user manual is now browsable without leaving Wallfacer:

- **Embedded markdown docs** with table of contents, reading order, and prev/next navigation
- **Mermaid diagram rendering** with pan/zoom viewer for architecture diagrams
- **Inter-document links** that navigate within the viewer
- **Fullscreen toggle** for deep reading sessions
- Auto-generated TOC with left-border active indicators

## 🎯 Task Goals & Specs — Separate *What* From *How*

Tasks now have a dedicated **Goal** field separate from the implementation spec:

- **Goal**: the human-readable objective ("Fix the login timeout bug")
- **Spec**: the detailed agent instructions (implementation details, constraints)
- Edit/Preview tabs with synced heights for both fields
- Goals are visible on cards, searchable, and used in oversight summaries

## 🔄 Dependency Graph & Batch Tasks

- **Visual dependency graph** with SVG layout and cycle detection
- **Batch task creation** with symbolic dependency wiring — create entire task trees atomically
- Dependencies visible on Kanban cards and in the detail modal
- Critical-path scoring drives auto-promotion order

## 🛡️ Circuit Breakers & Fault Isolation

The automation layer got serious about reliability:

- **Per-watcher circuit breakers** — a failing auto-test doesn't take down auto-promote
- **Container launch circuit breaker** — stops hammering a broken runtime
- **Visual circuit breaker health** in the UI — see exactly which watchers are healthy
- Replaced nuclear `pauseAllAutomation` with surgical per-watcher isolation
- Settle delays on auto-test and auto-submit to prevent rapid-fire triggers
- **Auto-promoter fills all available capacity in one pass** — no more one-at-a-time delays

## 🔁 Auto-Retry with Failure Categories

Failed tasks don't just sit there anymore:

- **Automatic failure categorization** — timeout, budget_exceeded, container_crash, agent_error, worktree_setup, sync_error
- **Per-category retry budgets** — transient failures get retried, permanent failures don't
- **Auto-retry watcher** with configurable budget and exponential backoff
- Manual retry resets counters; fresh session option for stuck tasks

## 📊 Sandbox Image Management

- **Image status API** — check which sandbox images are cached locally
- **Pull/delete from UI** — manage sandbox images directly from Settings
- **Streaming pull progress** with structured progress reporting
- Doctor command checks image availability on startup

## ⚡ Performance & Architecture

Major internal improvements:

- **Status-partitioned index** eliminates O(n) task scans
- **LRU diff cache** with immutable caching for completed tasks
- **Lazy event loading** for terminal-state tasks — startup is dramatically faster
- **Append-only log rendering** — no more full DOM rebuilds on every log line
- **Fused board context generation** with change-sequence caching
- **Deep-copy on all read paths** — no more data races from shared task references
- **Event compaction** bounded to session horizon
- **BroadcastChannel SSE sharing** across browser tabs
- **SSE heartbeat with staleness detection** — no more phantom disconnects
- **Merged `instructions` into `prompts`** — unified prompt architecture, one less package boundary

## Upgrading

### One-liner install (recommended)

```bash
curl -fsSL https://raw.githubusercontent.com/changkun/wallfacer/main/install.sh | sh
```

### Direct binary download

Download the binary for your platform from the **Assets** section below, then:

```bash
chmod +x wallfacer-*
sudo mv wallfacer-* /usr/local/bin/wallfacer
wallfacer doctor   # verify setup
wallfacer run      # launch
```

### From source

```bash
go install changkun.de/x/wallfacer@v0.0.6
```
