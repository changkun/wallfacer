# v0.0.4

## Release Summary

- **Release date:** 2026-03-08
- **Previous release:** `v0.0.3` (2026-02-22)
- **Range:** `v0.0.3..v0.0.4`
- **Delta:** **255 commits**, **155 files changed**, **+34,291 / -1,805**

## Why This Release Matters

This release spans roughly two calendar weeks, but the core architectural leap happened in about **two days of concentrated development**.

In that burst, wallfacer crossed a threshold from a tool-assisted workflow into a **self-developing system**: it now drives planning, implementation, testing, and operational feedback loops as a **fully automated engineering team**.

## Major Changes

### 1. Sandbox and Execution Architecture

- Shifted toward backend-agnostic, multi-sandbox orchestration.
- Added per-sandbox model routing and per-activity sandbox overrides.
- Unified container argument construction with `ContainerSpec`.
- Consolidated sandbox layout to `sandbox/{claude,codex}` and expanded Codex support.

### 2. Task Lifecycle Is Now Explicit and Enforceable

- Enforced strict task state transitions with transition tables.
- Added tested transition methods for task statuses.
- Improved correctness of waiting/committing/done flows.
- Hardened conflict/sync handling to prevent invalid regressions.

### 3. Autonomous Operations Became Much Stronger

- Added periodic ideation and oversight generation with scheduling controls.
- Improved automatic movement from brainstorm to in-progress and from verified states onward.
- Reduced manual intervention across orchestration paths.

### 4. Oversight, Traceability, and Telemetry

- Added execution-phase timing spans and aggregate span stats.
- Improved agent-monitor trace visibility and detail (including tool arguments).
- Added usage statistics dashboard.
- Separated implementation/testing live output more clearly.

### 5. UI Workflow Acceleration

- Major modal and task-detail refactor with tabbed panels.
- Added timeline/flamegraph/spans improvements plus deep-link support.
- Improved dependency-graph usability and rendering stability.
- Refined refinement UX with safer defaults and stronger locking.

### 6. Reliability and Security Hardening

- Fixed goroutine leaks and strengthened graceful shutdown behavior.
- Added mutex protections in critical store/runner paths.
- Expanded tests across handler/runner/store/frontend JS.
- Added SSRF protections for environment base URL and model gateway fetch paths.

## Operator Notes

- Migrate legacy Claude-specific model env naming to current `WALLFACER_*` model variables.
- Expect changed defaults/layout in refinement and monitor panels.
- Status transitions are stricter than `v0.0.3`, especially around waiting/testing/committing flows.
