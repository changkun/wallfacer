### What if your engineering team never stopped working?

Wallfacer v0.0.5 takes autonomous software engineering from "impressive demo" to **daily driver**. This release ships the features that make you trust the system enough to queue up 20 tasks, close your laptop, and come back to merged PRs.

---

## 🧠 Autonomous Ideation — Your AI Now Has Ideas

Wallfacer doesn't just execute your tasks anymore — it **thinks of new ones**.

- **Periodic brainstorming** generates task ideas ranked by impact score, automatically creating cards on your board
- Ideas flow from brainstorm → backlog → in-progress with zero manual intervention
- Configurable brainstorm intervals (default: 1 hour) — let it marinate or let it sprint
- Full prompt synthesis: the brainstorm agent reads your codebase, your AGENTS.md, and your existing tasks to propose what matters most

## 💰 Cost & Budget Controls — Spend Wisely, Ship More

You asked for guardrails. We delivered.

- **Per-task cost and token budget limits** — set a ceiling and the agent stops before blowing through it
- **Per-turn usage tracking** with cumulative cost overlays on the flamegraph and timeline
- **Usage analytics dashboard** with aggregate stats across all tasks
- Autopilot automatically disables on max-token hits — no runaway spending

## 📊 Timeline & Flamegraph — Finally, You Can *See* What Happened

The execution visualizations got a complete overhaul:

- **Non-linear time compression** — idle gaps between activity are compressed so you see the actual work, not the waiting. Hatched break indicators show where time was compressed, with duration labels and hover tooltips showing the full time range
- **Oversight phase distribution fix** — when the LLM generates poorly-timed oversight phases, they're now redistributed evenly so all phases are visible instead of one dominating 90% of the timeline
- **Activity-aware cost chart** — see exactly where money was spent across implementation, testing, oversight, and title generation turns
- **Span statistics** with visual breakdowns and percentile analysis

## 🔍 Full-Text Search Across Everything

- **Server-side search** with in-memory index — instant results across task titles, prompts, tags, and oversight summaries
- Zero per-query disk I/O — the search index stays hot in memory and updates on every mutation

## 🎨 Settings & UI Polish

The settings experience was rebuilt from the ground up:

- **Tabbed vertical layout** — clean, organized, no more endless scrolling
- **Embedded env configuration** — API keys, model selection, and all config in one place
- **Prompt templates system** — save and reuse prompt patterns across tasks
- **Paginated archived tasks** — boards with hundreds of completed tasks stay fast
- **Command palette** (`Ctrl+K`) — keyboard-first task actions, search, and navigation
- **@ mention autocomplete** with auto-focus and tab/enter selection
- Collapsible oversight accordions on task cards
- Gradient-styled priority and impact badges

## 🔧 Codex Sandbox Support

Wallfacer now runs OpenAI Codex alongside Claude Code:

- **Dual-sandbox orchestration** — route tasks to Claude or Codex based on model config
- **Automatic fallback** — when Claude hits token limits, Codex picks up the work
- Host-auth validation and cache fallback for Codex credentials
- Full oversight and timeline support for Codex execution streams

## 🏗️ Architecture & Reliability

Under the hood, this release is substantially more robust:

- **Strict state transitions** with trigger fields on every state change event
- **Schema versioning** for task.json with automatic migration
- **Monotonic delta replay** for SSE task streams — no more stale state on reconnect
- **Prometheus `/metrics` endpoint** for production monitoring
- **Cursor pagination** and type filtering for task events
- Centralized route registry (`apicontract`) — single source of truth for all HTTP paths
- Hardened JSON decoding to prevent parameter smuggling
- Fixed goroutine races in refinement, modal lifecycle, and container monitoring
- Golden-file regression suite for output parsing

## 📈 Testing & CI

- Unified Go + JS coverage reporting to Codecov
- Expanded frontend test coverage across markdown, mentions, env config, and API modules
- Stabilized flaky tests for command palette, refinement concurrency, and modal race conditions
- Coverage thresholds enforced in CI
